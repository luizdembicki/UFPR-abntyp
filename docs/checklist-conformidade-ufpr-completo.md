# Checklist de Conformidade UFPR/ABNT Completo

Checklist consolidado com base no Manual de Normalização de Documentos Científicos da UFPR (2022) e nas normas ABNT citadas no próprio manual.

Use este arquivo como validação de entrega do pacote, documento por documento. Sempre que o manual marcar algo como opcional, a presença do item depende do tipo de trabalho, do programa, do edital ou da mídia do documento.

## Regras transversais

### Formatação geral

- [ ] Papel A4.
- [ ] Margens conforme padrão institucional/ABNT adotado pelo pacote.
- [ ] Fonte padronizada no corpo do texto.
- [ ] Espaçamento 1,5 no texto principal.
- [ ] Recuo de primeira linha de parágrafo aplicado.
- [ ] Paginação contínua quando exigida.
- [ ] Numeração progressiva de seções.
- [ ] Títulos e subtítulos coerentes com o nível hierárquico.
- [ ] Sumário atualizado automaticamente.
- [ ] Referências ordenadas conforme ABNT.
- [ ] Figuras, tabelas e quadros com identificação, título, fonte e nota quando houver.

### Citações e referências

- [ ] Citações diretas curtas com aspas.
- [ ] Citações diretas longas em bloco destacado.
- [ ] Citações indiretas sem aspas.
- [ ] Sistema de chamada consistente no documento inteiro.
- [ ] Indicação de página em citações diretas quando aplicável.
- [ ] Entradas bibliográficas completas para todos os documentos citados.
- [ ] Títulos de obras destacados conforme o tipo de referência.
- [ ] Sem referências manuais fora do padrão do pacote quando existir .bib.

### Elementos gráficos e matemáticos

- [ ] Figuras com legenda, fonte e nota quando necessário.
- [ ] Quadros com moldura e legenda padronizada.
- [ ] Tabelas conforme norma tabular adotada.
- [ ] Equações numeradas quando citadas no texto.
- [ ] Fórmulas e símbolos explicados na primeira ocorrência.
- [ ] Títulos de figuras, quadros e tabelas com grafia padronizada.
- [ ] Separadores tipográficos consistentes, com traço espaçado quando o manual exigir.

### Lista pré-textual e pós-textual

- [ ] Capa.
- [ ] Folha de rosto.
- [ ] Resumo em português.
- [ ] Resumo em língua estrangeira.
- [ ] Palavras-chave e keywords.
- [ ] Lista de ilustrações quando houver ilustrações suficientes.
- [ ] Lista de tabelas quando houver tabelas suficientes.
- [ ] Lista de abreviaturas e siglas quando houver siglas e abreviaturas relevantes.
- [ ] Lista de símbolos quando houver símbolos relevantes.
- [ ] Sumário.
- [ ] Referências.
- [ ] Apêndices quando o autor produzir material complementar.
- [ ] Anexos quando houver material de terceiros.
- [ ] Glossário quando houver terminologia específica relevante.
- [ ] Índice quando houver necessidade de remissão alfabética/temática.

## Projeto de pesquisa / Pré-projeto

### Estrutura básica de projeto

- [ ] Capa quando exigida.
- [ ] Lombada ou dorso quando impresso e exigido.
- [ ] Folha de rosto.
- [ ] Lista de ilustrações quando aplicável.
- [ ] Lista de tabelas quando aplicável.
- [ ] Lista de abreviaturas e siglas quando aplicável.
- [ ] Lista de símbolos quando aplicável.
- [ ] Sumário.
- [ ] Introdução com tema, problema, hipóteses, objetivos e justificativa.
- [ ] Desenvolvimento com referencial teórico, metodologia, recursos e cronograma.
- [ ] Referências.
- [ ] Glossário quando aplicável.
- [ ] Apêndice quando aplicável.
- [ ] Anexo quando aplicável.
- [ ] Índice quando aplicável.

### Regras específicas

- [ ] O tema está claramente delimitado.
- [ ] O problema está formulado como pergunta.
- [ ] As hipóteses, quando existirem, são verificáveis.
- [ ] O objetivo geral está formulado no infinitivo.
- [ ] Os objetivos específicos detalham as etapas.
- [ ] A justificativa mostra relevância técnica, científica, institucional ou social.
- [ ] A metodologia informa onde, como, com o quê, com quanto e quando.
- [ ] Os recursos foram separados em humanos, materiais e financeiros.
- [ ] O cronograma está organizado em quadro ou equivalente.
- [ ] O pré-projeto, quando pedido por edital, segue as exigências explícitas do edital.

## Relatório técnico / científico / acadêmico

### Estrutura básica

- [ ] Capa quando exigida.
- [ ] Lombada ou dorso quando impresso e exigido.
- [ ] Folha de rosto.
- [ ] Errata quando necessária.
- [ ] Agradecimentos quando desejados.
- [ ] Resumo em língua vernácula.
- [ ] Lista de ilustrações quando aplicável.
- [ ] Lista de tabelas quando aplicável.
- [ ] Lista de abreviaturas e siglas quando aplicável.
- [ ] Lista de símbolos quando aplicável.
- [ ] Sumário.
- [ ] Introdução.
- [ ] Desenvolvimento.
- [ ] Considerações finais.
- [ ] Referências quando houver citações ou quando o relatório exigir.
- [ ] Glossário quando aplicável.
- [ ] Apêndice quando aplicável.
- [ ] Anexo quando aplicável.
- [ ] Índice quando aplicável.
- [ ] Formulário de identificação quando exigido pela instituição.

### Tipos de relatório do manual

- [ ] Relatório de viagem ou participação em evento: data, destino, duração e objetivo na introdução; participantes e atividades no desenvolvimento; avaliação crítica na conclusão.
- [ ] Relatório de estágio: descrição do local, atividades desenvolvidas, horas por atividade e aproveitamento final.
- [ ] Relatório de visita técnica: descrição do local, data e objetivos; fatos observados; resultados da observação.
- [ ] Relatório administrativo: objetivos e cronograma; tarefas realizadas com data, local, método e impedimentos; recomendações finais.
- [ ] Relatório progressivo: objetivos e cronograma; etapas executadas e por executar; sugestões e críticas finais.

### Regras específicas

- [ ] O nome da entidade autora aparece quando o relatório é institucional.
- [ ] O título e o subtítulo identificam claramente o relatório.
- [ ] A data segue o formato solicitado pelo órgão ou programa.
- [ ] O verso da folha de rosto traz equipe técnica ou dados catalográficos quando exigidos.

## Artigo científico

### Estrutura básica

- [ ] Título.
- [ ] Subtítulo quando houver.
- [ ] Autoria.
- [ ] Filiação institucional quando exigida.
- [ ] Resumo.
- [ ] Palavras-chave.
- [ ] Abstract ou resumo em língua estrangeira quando exigido pelo veículo.
- [ ] Keywords.
- [ ] Corpo do artigo.
- [ ] Referências.

### Regras específicas

- [ ] O texto segue a estrutura do periódico ou evento.
- [ ] As citações e referências obedecem ao padrão ABNT adotado pelo pacote.
- [ ] As seções internas seguem numeração progressiva quando o veículo permitir.
- [ ] Quando o periódico exigir, observar limites de palavras, seções e padronização editorial.

## Memorial

### Estrutura básica

- [ ] Identificação do autor.
- [ ] Trajetória acadêmica, profissional e intelectual.
- [ ] Organização cronológica ou crítica da experiência.
- [ ] Conexão entre formação, atuação e projeções futuras.
- [ ] Referências quando usadas.

### Regras específicas

- [ ] O texto apresenta reflexão crítica e não apenas currículo expandido.
- [ ] A narrativa mostra evolução de formação e produção.
- [ ] As fontes citadas, quando houver, aparecem nas referências.

## Currículo / Curriculum vitae / Lattes / ORCiD

### Estrutura básica

- [ ] Dados pessoais.
- [ ] Formação acadêmica.
- [ ] Formação complementar.
- [ ] Experiência profissional.
- [ ] Atuação docente quando aplicável.
- [ ] Representações, funções e filiações quando pertinentes.
- [ ] Produção científica e artística quando pertinente.

### Regras específicas

- [ ] O currículo para fins técnico-científicos segue o padrão exigido pela área.
- [ ] O Currículo Lattes é usado quando a instituição brasileira o exigir.
- [ ] O ORCiD é informado quando solicitado por edital, revista ou plataforma.
- [ ] O currículo profissional apresenta as informações de forma objetiva e enxuta.
- [ ] O documento pode ser usado para estágio, seleção, concurso ou consultoria.

## Livro / folheto / documento editorial

### Regras básicas

- [ ] Metadados editoriais completos.
- [ ] Autor ou organizador corretamente identificado.
- [ ] Título e subtítulo padronizados.
- [ ] Edição, local, editora e data quando aplicável.
- [ ] Referências usadas quando houver citações.

### Observações

- [ ] Verificar se o trabalho é livro completo, parte de livro, coletânea ou folheto.
- [ ] Conferir a norma específica do tipo documental quando houver publicação formal.

## Slides de defesa / apresentação

### Regras básicas

- [ ] Título da apresentação.
- [ ] Identificação do autor.
- [ ] Estrutura de tópicos clara.
- [ ] Citações quando houver suporte conceitual.
- [ ] Referências quando houver citações.

### Observações

- [ ] Não existe norma ABNT específica para slides.
- [ ] O pacote deve ser usado apenas nas partes de referência e citação quando necessário.
- [ ] A apresentação deve ser legível e consistente com o conteúdo oral.

## Elementos complementares do manual

### Redação técnico-científica

- [ ] Objetividade.
- [ ] Clareza.
- [ ] Imparcialidade.
- [ ] Uniformidade.
- [ ] Gramática revisada.
- [ ] Números, símbolos, datas e horas uniformes.
- [ ] Siglas explicadas na primeira ocorrência.
- [ ] Abreviaturas usadas de forma consistente.
- [ ] Remissivas e referências cruzadas verificadas.

### Acessibilidade informacional

- [ ] Figuras e imagens com descrição quando necessário.
- [ ] Texto alternativo quando a mídia permitir.
- [ ] Estrutura legível para leitores de tela quando possível.
- [ ] Contraste e organização visual adequados.

### Integridade acadêmica

- [ ] Fonte de todas as ideias de terceiros identificada.
- [ ] Evitar plágio literal e mosaico.
- [ ] Citações diretas e indiretas corretamente marcadas.
- [ ] Uso de materiais de terceiros autorizado quando necessário.
- [ ] Licenças Creative Commons verificadas quando houver reutilização.

## Critério final de aceitação

- [ ] O documento compila sem erros.
- [ ] O sumário reflete a estrutura real do trabalho.
- [ ] As listas pré-textuais aparecem quando necessárias.
- [ ] As referências saem do .bib com abnt.csl.
- [ ] O documento final bate com o tipo de trabalho escolhido.
- [ ] Não há inconsistências visuais entre títulos, listas e legendas.
